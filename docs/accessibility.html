Accessibility

Built from the ground up for users of all abilities

Accessibility First Design

VisionAssist is designed with accessibility as a core principle, not an afterthought. Every feature has been built to work seamlessly with assistive technologies and alternative input methods.

WCAG 2.1 AA Compliant

Meets Web Content Accessibility Guidelines for color contrast, focus indicators, and semantic structure.

Screen Reader Tested

Works with VoiceOver (iOS/macOS), NVDA (Windows), and TalkBack (Android).

Keyboard Navigable

Full functionality available without a mouse or touch screen.

Keyboard Shortcuts

Navigate and control the app entirely with your keyboard.

Key Action
1 Switch to Scene Description mode
2 Switch to Read Text mode
3 Switch to Object Identification mode
4 Switch to Navigation mode
5 Switch to Color Detection mode
Space Capture image (when camera is active)
C Open/close camera
V Activate voice command
R Repeat last description
S Stop speaking
H Open history panel
Q Open quick actions
Esc Close any open modal/panel
Tab Navigate between interactive elements

Voice Commands

Speak naturally to control the app hands-free.

Say This To Do This
"Capture" or "Take picture" Capture current camera view
"Scene" Switch to Scene Description mode
"Read" Switch to Read Text mode
"Identify" Switch to Object Identification mode
"Navigate" Switch to Navigation mode
"Color" Switch to Color Detection mode
"Stop" Stop current speech
"Repeat" Repeat last description
"History" Hear how many items are in history
"Last result" Repeat most recent analysis
"Continuous on" Enable auto-capture every 5 seconds
"Continuous off" Disable continuous mode
"Help" List available commands

Tip: Press V or tap the microphone button to start listening. The app will provide audio feedback when it's ready for your command.

Visual Settings

High Contrast Mode

Yellow text on black background for maximum visibility. Optimized for users with low vision or light sensitivity.

Toggle: Settings panel

Adjustable Font Size

Scale text from 14px to 28px to match your visual needs. Affects all content including results.

Range: 14px - 28px

Dark/Light Theme

Choose your preferred color scheme. Dark mode reduces eye strain in low light; light mode provides better visibility in bright environments.

Default: Dark mode

Focus Indicators

Clear, high-visibility focus outlines (3px cyan) on all interactive elements for keyboard navigation.

Audio Settings

Auto-Speak Results

Automatically read analysis results aloud. Toggle off if you prefer to use a screen reader.

Default: On

Sound Effects

Audio cues for capture, analysis complete, and errors. Provides non-visual feedback for actions.

Default: On

Haptic Feedback

Vibration patterns on mobile devices: short for capture, double for complete, long for errors.

Mobile only

Screen Reader Support

Tested With

  • VoiceOver - iOS and macOS
  • NVDA - Windows (free)
  • JAWS - Windows
  • TalkBack - Android

Implementation Details

  • ARIA Labels: All buttons and controls have descriptive labels
  • Live Regions: Status updates and results announced automatically
  • Semantic HTML: Proper heading hierarchy, lists, and landmarks
  • Focus Management: Focus moves logically when opening/closing panels
  • Skip Links: Hidden skip navigation for keyboard users

Tips for Screen Reader Users

  • Turn off "Auto-speak results" to avoid conflicts with your screen reader
  • Use keyboard shortcuts (1-5) to quickly switch modes
  • Press H to navigate by headings in results
  • The app announces when analysis is complete and results are ready

Mobile Accessibility

Large Touch Targets

All buttons are minimum 44x44 pixels for easy tapping. Capture button is extra large (80x80 pixels).

Gesture Support

Works with VoiceOver gestures on iOS and TalkBack gestures on Android.

Portrait & Landscape

Responsive layout adapts to any screen orientation and size.

Report Accessibility Issues

We're committed to making VisionAssist accessible to everyone. If you encounter any accessibility barriers or have suggestions for improvement, please let us know:

  • Open a GitHub Issue with the "accessibility" label
  • Describe the issue and which assistive technology you're using
  • Include steps to reproduce if applicable

We typically respond to accessibility issues within 48 hours.
